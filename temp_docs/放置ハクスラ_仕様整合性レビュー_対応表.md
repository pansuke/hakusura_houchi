# 3. 文書間の主な修正

| 旧記載 | 更新後 |
|---|---|
| 距離・座標を扱わない | 2D詳細座標は扱わず、レーン内の抽象1次元位置だけを扱う |
| 進軍は単一の支配度 | 各Characterの個別位置とPUSHで表現する |
| M3-BでLocal / Adjacent / Global | 対面・位置と同時に必要なためM3-Aへ移動 |
| M3-Bで本番最大Action | 本番最大Actionは廃止。Safety Limitはerror |
| shuffleなし / hand 5 | M1暫定。M3ではshuffle、initial 3、max 7 |
| 使用可能カードを先頭から探索 | M3では最古カードだけを判定する |
| 全滅勝敗 | M3-BではNexus破壊だけが勝敗条件 |
| 進軍専用Effect候補 | 専用分類は作らない |
| Global対象未定 | 生存敵Character全員、Nexus除外 |
| Adjacent対象なし未定 | 不発、Nexus除外 |

# 7. M4-A Product Owner Decision対応

| ID | 決定 |
|---|---|
| M4-001 | 各チームはTOP / MID / BOT / SUPPORTの4体編成とし、SUPPORTは1体配置する |
| M4-002 | 同一CharacterInstanceの重複配置は禁止。同じCharacterMasterの別Instanceは同時配置可能 |
| M4-003 | SUPPORTは通常配置時と同じDeckを使用する |
| M4-004 | SchedulerはTOP味方→TOP敵→MID味方→MID敵→BOT味方→BOT敵→SUPPORT味方→SUPPORT敵 |
| M4-005 | SUPPORTは通常の行動機会を持ち、支援要請0でもActionを省略しない |
| M4-006 | SUPPORTはMP / MPR / DS / Draw Gauge / Deck Runtimeを持つ |
| M4-007 | SUPPORTはHP / HPR / 位置 / PUSH / 対面 / 死亡 / 復活を持たない |
| M4-008 | 支援要請は各レーン0～9で保持する |
| M4-009 | 最大支援要請レーンを選び、同値または全0はBattle Seedで決定的にrandom選択する |
| M4-010 | 支援要請はカード使用条件ではなく、支援先選択の優先度である |
| M4-011 | 支援属性カードは記載された支援要請減少量だけ支援先の要請を減らす |
| M4-012 | 支援要請がカード記載値未満でもカードは使用でき、減少後は0へclampする |
| M4-013 | SUPPORTが通常カードを使用した場合、Damage / Heal / Draw Gauge増加を90%減少する |
| M4-014 | 通常カードを実際に使用した場合、選択した支援先レーンの支援要請を1減らす |
| M4-015 | カードを使用できなかった場合、支援要請は減らさない |
